Um die „XRio Converter“ - Datei weiter zu erforschen, wird durch Anklicken des Symbols „Import from list“ auf der Seite „Load XRio Converter and Rio file from list“ eine „XRio Converter“-Datei zum „AREVA MiCOM P441“ geladen. Nachdem die Datei „XRio Converter“ geladen wurde, ist es möglich, diese Datei separat aus der Software zu exportieren. Um dies zu tun, durch Klicken auf „Datei“ Menü, können Sie die „XRio Converter“ - Datei in Form von mit oder ohne Formel ausführen. Zum besseren Verständnis exportieren Sie in diesem Teil zunächst die Datei „XRio Converter“ ohne Formel mit der Option „Export without formula…“.

Dann wird die gleiche „XRio Converter“-Datei ohne Formel in die Software geladen. Wenn Sie die Datei "XRio Converter" in notepad++ öffnen, sehen Sie, dass diese Datei in Form einer "xml"-Datei geschrieben ist. In der Teilmenge von "XRio Converter" werden die Version und die Sprache der Datei definiert. Es wird dann beobachtet, dass die „XRio Converter“ - Datei ohne Formel aus zwei Hauptteilen von „Rio Converter“ und „Rio Type“ besteht.

Wenn Sie den Abschnitt „Rio Converter“ öffnen, können Sie sehen, dass dieser Abschnitt aus einem anderen Abschnitt mit dem Namen „Properties“ besteht, in dem Informationen über Datum und Uhrzeit erwähnt werden. Tatsächlich ist der Stil dieser Art von Codierung „xml“, wobei jeder Abschnitt mit einem Tag geöffnet und geschlossen wird.Oder, als ein anderes Beispiel, ist es möglich, das „Rio Converter“-Tag zu öffnen oder zu schließen, indem Sie auf das Quadrat daneben klicken. Das Tag „ScriptReference“ wird in derselben Zeile geöffnet und geschlossen, da am Ende ein „/>“-Zeichen vorhanden ist. Der zweite Teil, der das "Rio Type"-Tag ist, enthält feste Informationen des Relais und ist die gleiche "Rio"-Datei wie zuvor beschrieben.

Beim Öffnen dieser Registerkarte wird beobachtet, dass das Tag „Rio“ aus Blöcken „DEVICE“, „DISTANCE“, „OVERCURRENT“, „CBCONFIGURATION“ besteht, die aus den Relaisspezifikationen stammen. Diese Blöcke sind das "xml"-Modell der Datei "Rio", die in der Datei "XRio Converter" enthalten sind. Wie es geschrieben steht, ist das „Rio“ -Tag von der Art „Block“. Durch das Öffnen dieses Blocks wird beobachtet, dass der Block „Rio“ aus anderen Blöcken besteht. By opening the first tag it is observed that this tag, whose name is “DEVICE” is, too, of the “Block” kind. Inside this block there is a “NAMEPLATE” which is, also, of the “Block” kind. Beim Öffnen des ersten Tags wird festgestellt, dass dieses Tag, dessen Name „DEVICE“ ist, auch der Art „Block“ angehört. Durch das Öffnen dieses Blocks wird beobachtet, dass sich innerhalb dieses Blocks ein „DEVICEMODEL“ befindet, das der Art „Parameter“ entspricht. Dies bedeutet, dass dieser block einen „value“ von „P441 P442 P444“ hat. Ebenso gibt es andere Parameter wie „MANUFACTURER“, „DEVICETYPE“ usw. im Block „NAMEPLATE“.

Diese Informationen können in der Software „Vebko AMPro Test“ nach Eingabe der Datei „XRio Converter“ beobachtet werden. Die Informationen zur Datei “XRio Converter“ sind im Fenster „Testobjekt“ verfügbar. Im Baumdiagramm dieses Abschnitts befindet sich der Block „XRio Converter“ und seine Teilmengen wie der Block „Rio“. Ebenso ähneln der Block „Device“ und seine Teilmengen dem Block „Name Plate“. Im Block „Name Plate“ werden die in „XRio“ aufgelisteten Parameter in die Tabelle eingetragen. Beispielsweise wird der parameter „DEVICE MODEL“, der den Wert von „P441 P442 P444“ hat, in der Spalte „Value“ dieser Tabelle angezeigt. Als weiteres Beispiel wird im Block „Rio“ in der Datei „XRio Converter“ und im Pfad der Blöcke „DISTANCE“ und „PROTECTEDOBJECT“ beobachtet, dass der Wert von „LINE“ für „PT connection“ mit „ID“:“PTCONN“ eingegeben wird.

Wenn Sie denselben Pfad im Fenster „Testobjekt“ in der software „Vebko AMPro Test“ verfolgen, nämlich das Verzeichnis „Distance“ im block „Rio“, können Sie in der Tabelle „Protected Object“ sehen, dass für einen Parameter mit „ID“:“PTCONN“ in der Spalte „Value“, „at line“ eingetragen ist, während in der Datei „XRio Converter“, „Line“ als Wert eingegeben wurde. Der Grund für den Unterschied in den Werten der Software und der Datei „XRio Converter“ ist, dass die im Abschnitt „Rio“ geschriebenen Codes die Werte von „XRio“ empfangen und dann die entsprechenden Werte für verschiedene Parameter entsprechend dem Design der Codes eingeben, die in hier entspricht „Line“ für einen Parameter mit „ID“:“PTCONN“ im „XRio“ „at line“ in „Rio“.

In der Software „Vebko AMPro Test“ gibt es Spalten wie „Name“, „Description“ usw. für jeden Parameter, der dem Benutzer Informationen über den beabsichtigten Parameter liefert, und es möglicherweise keinen Code für diese Abschnitte in der Datei „XRio Converter“ gibt. Die Initialisierung dieser Spalten im „Rio“ ist so, dass nach dem vom Programmierer geschriebenen Code, wo immer in der Spalte „ID“ der Ausdruck „PTCONN“ geschrieben ist, sein Name „PT Connection“ ist und „Potential transformer connection side“ als Beschreibung in der Spalte „Description“ dafür eingetragen ist. Auch der Designer von anderen Spalten wie „Type“ hat es im Code erwähnt, der im Abschnitt „Rio“ angezeigt werden soll. Für den genannten Parameter ist es „Enumeration“, was bedeutet, dass es selektiv ist und der Benutzer einen Wert aus dem Dropdown-Feld in der Spalte „Value“ auswählen kann.

Es sollte beachtet werden, dass für jede Information im Teil „Custom“, die nicht festgelegt ist, sein Code in „XRio ” geschrieben werden muss. Dies wird in zukünftigen Videos mehr erklärt werden. Ein weiterer Punkt ist, dass der Modus „User Interface“ im Abschnitt „Rio“ enthalten ist. Dies bedeutet, dass durch doluble-Klick auf einen beliebigen Block der Teilmengen von „Rio“, z.B. „Device“ Block, die Seite „Device Setting“ öffnet sich, auf der alle Parameter der Teilmenge dieses Blocks beobachtet und geändert werden können. Wenn Sie jedoch auf einen anderen Block als „Rio“ doppelklicken, werden eine Reihe von Informationen über „ID“ des Blocks sowie dessen Namen angezeigt.
